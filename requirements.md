# 要求仕様書：macOS アプリケーション別音声出力切り替えアプリ

## 1. 概要

本ドキュメントは、Apple M4 Mac mini を主な対象とした、アプリケーションごとに音声出力デバイスを切り替えられる macOS ネイティブアプリケーションの要求仕様を定義する。

---

## 2. 目的

macOS は標準で「システム全体の出力デバイス」しか切り替えられない。本アプリケーションは、アプリケーション単位での音声ルーティングを実現し、例えば「音楽再生はスピーカー、ビデオ会議はヘッドフォン、ゲームは HDMI 接続のディスプレイスピーカー」といった柔軟な使い方を可能にする。

---

## 3. 対象環境

| 項目 | 内容 |
|------|------|
| ターゲットハードウェア | Apple Mac mini (M4 チップ) |
| OS バージョン | macOS Sequoia (15.0) 以降 |
| アーキテクチャ | Apple Silicon (arm64) |

---

## 4. 機能要件

### 4.1 アプリケーション別音声出力切り替え

| ID | 要件 |
|----|------|
| F-01 | 実行中のアプリケーションの一覧をリアルタイムで表示できること |
| F-02 | 各アプリケーションに対して、任意の音声出力デバイスを個別に割り当てられること |
| F-03 | 割り当ては即時反映され、アプリケーションを再起動せずに適用されること |
| F-04 | アプリケーション起動時に、保存済みの設定（プロファイル）が自動適用されること |
| F-05 | 出力デバイスとして「システムデフォルト」を選択できること |

### 4.2 HDMI パススルー対応

| ID | 要件 |
|----|------|
| F-06 | HDMI 経由で接続された外部ディスプレイのスピーカー／音声出力デバイスを認識できること |
| F-07 | Dolby Atmos・DTS など、HDMI パススルー対応のオーディオフォーマットを維持してルーティングできること |
| F-08 | HDMI デバイスの接続・切断を検出し、UI に即時反映すること |
| F-09 | HDMI デバイスが切断された場合は、フォールバックデバイス（例：内蔵スピーカー）へ自動的に切り替えること |

### 4.3 ユーザーインターフェース

| ID | 要件 |
|----|------|
| F-10 | メニューバー常駐型アイコンから素早くアクセスできること |
| F-11 | 実行中アプリ一覧と各アプリの現在の出力デバイスを一画面で確認できること |
| F-12 | ドラッグ＆ドロップ不要で、ドロップダウン選択のみで切り替えが完了すること |
| F-13 | アプリケーションアイコン・名称を表示し、どのアプリかを直感的に識別できること |
| F-14 | 設定プロファイルを複数保存・切り替えできること |
| F-15 | ダークモード／ライトモード双方に対応したデザインであること |

---

## 5. 非機能要件

| ID | 区分 | 要件 |
|----|------|------|
| NF-01 | パフォーマンス | 音声ルーティング切り替えのレイテンシが 200ms 以内であること |
| NF-02 | パフォーマンス | メモリ使用量が通常動作時に 50MB 以下であること |
| NF-03 | 信頼性 | macOS アップデート後も機能が継続して動作すること（パブリック API のみ使用） |
| NF-04 | セキュリティ | Apple の App Sandbox ガイドラインに準拠すること |
| NF-05 | セキュリティ | マイクアクセス等の不要な権限を要求しないこと |
| NF-06 | ユーザビリティ | 初回起動時にオンボーディングガイドを表示すること |
| NF-07 | ユーザビリティ | 操作ステップ数が 3 ステップ以内で任意のアプリの出力先を変更できること |
| NF-08 | 保守性 | ソースコードに対してユニットテストカバレッジ 70% 以上を維持すること |

---

## 6. 制約事項

- Apple の公開 API（CoreAudio、AVFoundation 等）の範囲内で実装すること
- プライベート API・未公開 API は使用しないこと（App Store 審査対応）
- macOS のサンドボックス制限により、他プロセスの音声ストリームに直接介入することは行わない。代わりに CoreAudio の仮想オーディオデバイス（Audio Unit / AudioServerPlugin）を用いてルーティングを実現する

---

## 7. 用語定義

| 用語 | 定義 |
|------|------|
| 音声出力デバイス | スピーカー、ヘッドフォン、HDMI 接続先など、macOS が認識する物理または仮想の音声出力先 |
| HDMI パススルー | 音声を macOS でデコードせず、接続先デバイス（AVアンプ等）に生ストリームとして転送する方式 |
| プロファイル | 複数のアプリ→デバイス割り当て設定をまとめて保存・呼び出しできる単位 |
| 仮想オーディオデバイス | CoreAudio の AudioServerPlugin として実装される、OS レベルの音声ルーティング中継デバイス |

---

## 8. 将来拡張（スコープ外）

- AirPlay デバイスへの音声ルーティング
- Bluetooth デバイスのコーデック（AAC/aptX）選択
- 複数出力への同時ルーティング（ミラーリング）
- iOS / iPadOS との連携（Continuity Audio）
